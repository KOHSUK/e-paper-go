# リポジトリガイドライン

## プロジェクト概要
`e-paper-go` は Raspberry Pi などから e-paper ディスプレイを扱うための Go ライブラリです。現在は Waveshare 2.13inch e-Paper B V4 向けのドライバと、e-paper 用の簡単な描画ヘルパを提供しています。

## 構成
- `epd2in13bv4/`: Waveshare 2.13inch e-Paper B V4 のドライバ、バッファ変換、表示制御。
- `render/`: `image.RGBA` キャンバス作成、フォント描画、罫線描画などの汎用描画ヘルパ。
- `README.md`: 利用例と公開 API の概要。

アプリ固有の天気、時計、ゴミ収集予定などのロジックはこのリポジトリに入れず、利用側アプリケーションに残してください。

## 開発コマンド
- `go test ./...`: 全パッケージのテストを実行。
- `go test ./epd2in13bv4`: 画像バッファ変換など、ハードウェアなしで検証できる部分を実行。
- `go mod tidy`: 依存関係を整理。
- `gofmt -w <files>`: 変更した Go ファイルを整形。

環境によって Go の build cache が書き込めない場合は、`GOCACHE=/tmp/codex-go-cache go test ./...` のように書き込み可能な cache を指定してください。

## コーディング方針
Go コードは常に `gofmt` 済みにしてください。パッケージ名は小文字にし、公開 API は利用側のアプリから本当に必要なものに絞ります。ハードウェア制御コードでは、失敗を握りつぶさず `error` として返す方針を維持してください。

表示サイズ、SPI 設定、GPIO ピンなどの定数は、可能な限り `Config` や公開定数で扱い、利用側が wiring を差し替えられるようにします。

## テスト方針
ハードウェアに依存しない処理は単体テストを追加してください。特に画像から 1-bit buffer への変換、サイズ検証、座標回転、ink count のような純粋関数はテスト対象です。

実機が必要な変更は、テストだけで確認済みにせず、PR や commit message に Raspberry Pi と対象 e-paper モジュールでの手動確認内容を書いてください。

## コミット方針
コミットメッセージは短い命令形の英語を基本にします。例: `Add display buffer tests`。公開 API、配線のデフォルト値、表示シーケンスを変える場合は、本文に互換性への影響を書いてください。
